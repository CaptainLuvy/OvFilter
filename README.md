# OvFilter

OvFilter is an overlap filtering tool specifically designed for long-read (HiFi / ONT) sequencing data. It effectively removes false-positive overlaps while maximizing the retention of true biological alignments by combining **low-frequency k-mer based collinearity checks** with a **Repeat Penalty Score (RPS) rescue mechanism**.

## Installation

OvFilter is written in C++ and requires `zlib`. To compile the program from source, simply clone the repository and run `make`, or compile it manually using `g++`:

```bash
git clone https://github.com/YOUR_USERNAME/OvFilter.git
cd OvFilter

# Compile using g++
g++ -O3 -o ovfilter ovfilter.cpp -lz -lpthread
```

## Quick Start

```bash
./ovfilter --paf input.paf \
           --reads reads.fa \
           --unikmers unikmers.txt \
           --output filtered.paf \
           --read-type hifi
```

### Required Parameters
- `--paf` / `-p`: Raw overlap file (e.g., `.paf` generated by minimap2).
- `--reads` / `-r`: FASTA file containing the sequence information.
- `--unikmers` / `-u`: Text file containing extracted low-frequency k-mers (one per line).
- `--output` / `-o`: Filtered output file.
- `--read-type`: Data type, either `hifi` or `ont`. The program will automatically adapt to the optimal default parameters.

### Optional Parameters
- `-k`: k-mer length (Default: 21).
- `-s` / `--min-shared`: Minimum shared unikmer count (Default: HiFi 15, ONT 6).
- `-i` / `--min-identity`: Minimum identity threshold.
- `-R` / `--rps-threshold`: RPS rescue threshold (Default: 0.08).
- `--keep-overhang`: Enable strict overhang filtering (Default: disabled, does not drop overlaps due to overhangs).

## Pre-requisite: How to Extract Unikmers

OvFilter requires a pre-extracted list of unikmers (`--unikmers`). The extraction method depends on whether you are running an All-vs-All (AVA) assembly or a Read-to-Reference (MAP) evaluation.

### Scenario 1: AVA Mode (All-vs-All Assembly)

In AVA mode, reads have sequencing depth (e.g., 30X), and the true biological k-mer frequencies form a normal distribution. We need to extract k-mers within this distribution window.

Use the provided helper scripts in the root directory for automated extraction:
1. **Install dependencies**: `conda install -c bioconda jellyfish -c conda-forge xlsxwriter`
2. **Configure**: Open `extractUnikmers.sh` and change the `READ_FILE` variable to your input reads path.
3. **Run**:
   ```bash
   bash extractUnikmers.sh
   ```
   *The script will automatically count k-mers, calculate the distribution window, generate a visualization Excel file, and export the final `unikmers.kmers` list.*

### Scenario 2: MAP Mode (Read-to-Reference)

When mapping reads to a reference, the reference depth is effectively 1. We simply need to extract unique k-mers from the reference genome.

**Step 1: Extract unique k-mers from Reference only**
For both HiFi and ONT data, it is highly recommended to extract strictly unique k-mers (appearing exactly once):
```bash
jellyfish count -m 21 -C -s 1G -t 16 reference.fasta -o ref_counts.jf
jellyfish dump -c -U 1 ref_counts.jf | awk '{print $1}' > map_unikmers.txt
```

**Step 2: Combine sequences to run OvFilter (Important!)**
In MAP mode, the Query is the Read and the Target is the Reference. OvFilter needs to access the sequences of both to perform collinearity checks. Therefore, you **must combine the reads and the reference** into a single file before passing it to `--reads`:
```bash
cat query_reads.fa reference.fa > combined.fa
./ovfilter --paf map.paf --unikmers map_unikmers.txt --reads combined.fa --read-type ont --output filtered.paf
```

## Output & Logs
Upon completion, the program will print a filtering summary to the terminal and generate a `.stats.txt` file in the output directory. This file details the number and percentage of overlaps discarded for various reasons (e.g., self-overlaps, contained, collinearity failure).
